# AiRFLOW 空气净化器 — 硬件接线与烧录指南

> 基于 ESP32-S3-DevKitC · 10×10cm 洞洞板 · 2.54mm 间距

---

## 一、硬件架构总览

| 模块 | 型号/规格 | 板载位置 |
|------|-----------|----------|
| 主控 | ESP32-S3-DevKitC | 左中，2×20P 排母对插 |
| 电机 | 43F704S100 (6P) | 右侧 |
| 电平转换 | 4ch (3.3V↔5V) | 主控与电机之间 |
| 温度传感器 | DS18B20 (TO-92) | 左下 |
| 音频功放 | PAM8302 (5P) | 中下 |
| 通信模块 | Y01 4P (UART) | 右下 |
| 板载LED | WS2812 (GP38) | ESP32 板载 |

### 供电架构

```
12V ──→ 电机功率 (RED)
        └── 470µF/25V 电解 → GND

5V  ──→ PAM8302 (+5V)
        ├── Y01 (VCC)
        ├── 电平转换 HV (VCC)
        └── 电机 (ORG, 逻辑)

3V3 ──→ ESP32 (3V3)
        ├── DS18B20 (VDD)
        ├── 电平转换 LV (VCC)
        ├── R3 (DQ 上拉)
        └── R4 (FG 上拉)

GND ──→ 全部模块互通（万用表蜂鸣档确认）
```

| **3V3** | 🔵 蓝 | 电源 → ESP32 · 电平LV · DS18B20 · R3/R4 上拉 | — | 主控+传感器+R3+R4 |
| **GND** | ⬛ 黑 | 全部模块 GND 互通 | — | 万用表蜂鸣档确认 |

| 网络 | GPIO | 颜色 | 连接路径 | 无源器件 | 说明 |
|------|------|------|----------|----------|------|
| **CLK** | GPIO39 | 🩵 青 | GP39 → 电平LV1 · 电平HV1 → 电机 WHT | — | 3.3V↔5V 电平转换 |

### 布局要点

- **电源总线**：板子上方横向走 12V / 5V / 3V3 / GND 四根总线
- **信号走线**：所有信号线在模块丝印外侧走线，避免交叉干扰

- **LCD 直插**：LCD 通过排针直接对插到 ESP32 排母上

## 五、固件烧录方法

### 5.1 准备工作

1. 用 USB-C 数据线连接 ESP32-S3-DevKitC 到电脑
2. 确认设备管理器中出现 COM 口（如 `COM3`、`COM4` 等）
3. 确保 `esptool.exe` 和 `firmware.bin` 在 `flash.bat` 同目录下

### 5.2 使用 flash.bat 一键烧录

直接双击运行 `flash.bat`，脚本会自动：

1. 检测可用 COM 口
2. 以 921600 波特率烧录固件
3. 烧录完成后自动复位 ESP32

```
双击 flash.bat → 自动检测串口 → 自动烧录 → 完成!
```

### 5.3 手动烧录

如果自动检测失败，可手动指定端口：

```bash
esptool.exe --chip esp32s3 --port COM3 --baud 921600 \
  --before default_reset --after hard_reset write_flash \
  -z --flash_mode dio --flash_size 16MB 0x0 firmware.bin
```

> 将 `COM3` 替换为实际的串口号。

### 5.4 烧录参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `--chip` | esp32s3 | 芯片型号 |
| `--port` | COMx | 串口号（自动检测） |
| `--baud` | 921600 | 烧录波特率 |
| `--flash_mode` | dio | Flash 模式 (DIO) |
| `--flash_size` | 16MB | Flash 容量 |
| `0x0` | — | 烧录起始地址 |

### 5.5 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `[错误] 未检测到串口!` | USB 未连接或驱动未安装 | 检查 USB 线、安装 CH340/CP2102 驱动 |
| `[错误] 烧录失败!` | COM 口被占用（如串口监视器） | 关闭其他串口工具后重试 |
| 乱码 / 命令无法识别 | bat 文件编码问题 | 确认文件为 UTF-8 BOM + CRLF 格式 |
| `esptool.exe` 不存在 | 工具未安装 | 从 ESP-IDF 或 Espressif 官网获取 |

### 5.6 flash.bat 编码说明

`flash.bat` 使用以下格式确保 Windows 中文控制台正常运行：

- **编码**：UTF-8 with BOM（`EF BB BF`）
- **换行符**：CRLF（`0d 0a`）
- **代码页**：脚本第 2 行 `chcp 65001` 切换到 UTF-8

> ⚠️ 如果在编辑器中修改 `flash.bat`，请确保保存为 **UTF-8 with BOM** + **CRLF 换行符**，否则会出现乱码或命令解析错误。
